# Humanize + slim the framework's instruction docs

## Goal

Make the framework's human-facing docs readable for a non-programmer, drop legacy/filler, slim the README
to a landing page with a one-line install, and move install detail to a linked page.

## Did

- Ran `/v-team`. Scoped via approval-gate questions to: human-facing docs + a light spec tidy; heavy
  README cut with link-outs.
- Rewrote `README.md` (297 → ~70 lines): plain "what it is", one-line install, attach-to-project, a brief
  command table, "learn more" links. Added a "before you start" prerequisite line. Dropped the repeated
  "no submodule / never vendored" asides.
- Created `INSTALL.md` — the detailed install reference absorbed from the README (flags table, sudo and
  Python-3.10+ notes, `install.sh` refresh, uninstall, the two Docker test tiers).
- Humanized `vault-guide.md` — reworded prose (em-dashes, stacked parentheticals, mechanical bold) while
  keeping every section, table, and code block intact.
- Light passes: `tool-playbook.md` (prose only), `_moc.md` (added missing `/v-do`, `/v-ask`, `/v-cr`),
  `VAULT.md` + `templates/VAULT.md` (tightened wordiest comments, no keys changed), `commands/v-init.md`
  (dropped "preferred for now").
- Ran a 3-critic diff-review panel (technical-accuracy, AI-tell audit, non-programmer clarity).
- Committed on branch `docs/humanize-instructions` (commit 9109414).

## Learned

- My `Write` calls leaked a literal `</content>` line into all three new/rewritten files (README,
  INSTALL, vault-guide). The AI-tell auditor caught it; `grep` confirmed it. Worth a grep guard after any
  large `Write`.
- `graphifyy` (double-y) is the real pipx package name — not a typo, despite reading like one. The
  accuracy guardian verified it against `setup.sh`.
- The spec files (`commands/**`, `personas/**`, `templates/**`) are already clean: nearly every HTML
  comment is load-bearing (template placeholders, `v-cr` fingerprint markers, persona citations). A
  "humanize everything" pass there would mostly be churn or risk.

## Behaviors & rules

- README is a landing page: what-it-is + one-line install + attach + command table + links; deep detail
  lives in linked pages (`INSTALL.md`, `vault-guide.md`), not inline.
- Humanize prose, but leave reference tables and code blocks dense — judge prose for AI tells, not tables.
- After a large `Write` of Markdown, grep for stray tooling artifacts (e.g. `</content>`) before commit.
