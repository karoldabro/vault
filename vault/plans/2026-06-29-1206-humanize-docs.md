# Plan — Humanize + de-clutter the framework's instruction docs

## Goal

Make the framework's human-facing docs readable for a non-programmer, drop legacy/filler, and turn the
README into a short landing page with a one-line install that links out for detail. No behavior change.

## Decisions (from approval-gate clarifications)

- Scope: **human-facing docs + a light spec tidy** (dead comments / obvious filler only, no spec restructuring).
- README: **heavy cut, link out** — move install detail to a dedicated page, keep README as a landing page.

## Audience split

- **Human-facing** (rewrite): `README.md`, `vault-guide.md`, `tool-playbook.md`, `_moc.md`, `CLAUDE.md`,
  `VAULT.md` + `templates/VAULT.md` comments.
- **Claude-facing specs** (light tidy only): `commands/**`, `personas/**`, `templates/**`, `prompts/**` —
  dense on purpose; almost every HTML comment is load-bearing. Touch only true filler.

## Steps

1. **Create `INSTALL.md`** (new detailed page). Absorbs from README: the flags table, the sudo/not-sudo
   note, the Python ≥3.10 note, `install.sh` refresh, uninstall, and the two test tiers. Written plainly
   but complete. This is the "detailed page" the README links to.

2. **Rewrite `README.md`** → landing page (~90 lines from 297). Sections:
   - One short plain-language paragraph: what the vault is and who it's for.
   - **Install** — one copy-paste line + "what it does" in two sentences; link to `INSTALL.md` for flags/options.
   - **Attach to a project** — the `vault-init.sh` one-liner (+ `--in-repo`), one sentence on what it sets up.
   - **Commands** — the brief table (keep), link to `vault-guide.md` §11 for the full reference.
   - **Learn more** — links to `vault-guide.md`, `tool-playbook.md`, `INSTALL.md`.
   Drop from README: contents tree, path-resolution table, `/v-cr --sandbox` deep-dive (link to it),
   required-tools deep-dive (one-line mention + link). Remove the repeated "no submodule / never vendored"
   asides.

3. **Humanize `vault-guide.md`** — keep the structure and reference tables (they earn their length), but
   thin the prose: cut em-dash pile-ups, collapse stacked parentheticals, drop the repeated submodule
   reminders, de-bold mechanical emphasis. No section removed; this stays the canonical process doc.

4. **Humanize `tool-playbook.md`** — light pass: trim em-dashes/bold, keep all rules + examples verbatim
   in meaning (they're load-bearing). Lowest-risk file.

5. **Fix + humanize `_moc.md`** — refresh the stale "Sessions (recent)" list, add the missing commands
   (`/v-ask`, `/v-do`, `/v-cr`) to the Commands list, light prose pass.

6. **`CLAUDE.md`** (repo) — already short; leave as-is or one-line tidy. No structural change.

7. **`VAULT.md` + `templates/VAULT.md`** — keep every config key and its explanatory comment (they're the
   docs for the format), but tighten the wordiest comment lines. No key added/removed.

8. **Light spec tidy** — remove obvious filler only (e.g. `v-init.md` "preferred for now"). Leave all
   template placeholders, v-cr fingerprint markers, persona citations, and behavioral prose untouched.

## Test / verification plan (docs-only)

- No code paths change. Verification is non-broken-reference + factual-accuracy:
  - Every relative link in README/guide/_moc resolves to a real file (grep + `test -f`).
  - The one-line install command is byte-accurate against current `setup.sh` flags.
  - No command name, flag, path, or MCP tool name altered vs. the originals (diff the factual claims).
  - `git grep` shows no remaining dangling references to anything moved to `INSTALL.md`.

## Diff-review panel (EXECUTE)

Light doc panel, run on the rewrite diff (not the plan):
- **technical-accuracy-guardian** — every command/flag/path/fact still true; nothing invented.
- **ai-tell-auditor** — humanizer pass: em-dashes, rule-of-three, copula avoidance, promotional words, fragments.
- **non-programmer-reader** — can someone non-technical install + attach without prior context.

## Open trade-offs

- Moving install detail to `INSTALL.md` means two files to maintain. Accepted: keeps README a true landing page.
- `vault-guide.md` stays long by design — it's reference, not a tutorial. Humanized, not shortened.
